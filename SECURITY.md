# Security Policy

## Supported Versions

Currently, only the latest version of OmniCode is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |

## Reporting a Vulnerability

We take the security of OmniCode seriously. If you believe you have found a security vulnerability, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please send an email to security@example.com (replace with a real email if applicable).

### Our Process

1.  We will acknowledge receipt of your report within 48 hours.
2.  We will investigate the issue and confirm the vulnerability.
3.  We will work on a fix and determine a release schedule.
4.  We will notify you once the fix is released.

### What we ask of you

*   Provide detailed steps to reproduce the vulnerability.
*   Allow us a reasonable amount of time to fix the issue before making any information public.
*   Do not perform any actions that could harm our users or the stability of the project.
